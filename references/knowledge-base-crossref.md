# 知识库交叉引用协议

> 共享参考文件——各插件在涉及法律知识库检索时引用本协议。
> 本文件是知识库对外部消费者的强制接口契约。

---

## 强制查找链（四步协议）

法律知识检索按以下四步依次执行，不得跳过：

### Step 1：路由规则加载

先读取知识库的路由配置，获取效力分级、源发现机制和检索策略。路由配置定义了哪些来源优先、哪些来源需要效力警示、以及各类型法律问题的推荐检索路径。

关键信息：
- 优先源列表（理解与适用系列、类案指南、最高院审判实务、案例库资源、权威问答选登）
- 扩展源列表（实务文章、地方审判指引、权威学术著作）
- 警示源列表（需核实现行效力的来源）
- 按法律领域的专项检索路径

### Step 2：概念体系检索（零 API 成本，最快速）

优先检索知识库中的概念文章（Wiki/Concepts 层），获取经过深度整理和综合分析的体系化知识。

**适用场景**：概念性问题（"什么是..."）、体系化知识、已有深度整理的概念。

**阅读协议**：
1. 先看概念文章的成熟度分级（T1-T4）——判断结论的可靠程度
2. 再看"来源权威分布"块——判断该文结论的权威基础
3. 若文章大量依赖效力待核实来源，或成熟度为 T3/T4，须进入 Step 3 交叉验证

**输出标注**：引用概念文章时标注 `[知识库概念 — YYYY-MM-DD]`，注明文章路径和最后更新日期。

### Step 3：原始数据源检索（优先源 → 扩展源）

概念体系不足时，按路由配置的优先级检索原始数据：

```
优先源（理解与适用、类案指南、最高院审判实务、案例库资源、权威问答选登）
  ↓ 不足
扩展源（实务文章 → 权威学术著作 → 地方审判指引）
  ↓ 仍不足
外部补充（MCP 检索、联网搜索）
```

**输出标注**：
- 优先源结果：标注 `[优先源：{源名称}]`
- 扩展源结果：标注 `[扩展源：{源名称}]`，附"可作为实务参考但不构成正式法律依据"
- 效力警示源：必须附 `[⚠需核实现行效力]` 提示，不得作为法律结论的唯一依据

### Step 4：外部补充（最后手段）

仅当知识库所有层级都无法回答时，才启动 MCP 检索（如 yuandian、北大法宝）或联网搜索。

**输出标注**：使用外部来源时标注 `[yuandian检索]`、`[联网检索 — 需复核]` 或 `[模型知识 — 需验证]`。

**升级钩子**：如四步协议全部走完后核心问题仍无可靠依据，或产出中 ≥2 处标注了 `[需验证]`/`[需复核]`，考虑启动更深度的多源并行检索。

## 来源标注速查

| 来源 | 标签 | 说明 |
|------|------|------|
| 知识库概念文章 | `[知识库概念 — YYYY-MM-DD]` | 注明文章路径和更新日期 |
| 理解与适用 | `[优先源：最高法释义]` | 高权威性 |
| 案例库资源 | `[优先源：案例库]` | 高权威性 |
| 权威问答选登 | `[优先源：官方答复]` | 高权威性 |
| 法院审判指导 | `[⚠需核实现行效力]` | 切忌作为唯一依据 |
| 公众号/实务文章 | `[扩展源：{来源名称}]` | 代表作者观点，需结合正式法源验证 |
| yuandian MCP 检索 | `[yuandian检索]` | 需复核 |
| 联网搜索 | `[联网检索 — 需复核]` | 未经二次验证 |
| 模型知识 | `[模型知识 — 需验证]` | 默认标签，未独立核实 |

---

## 约束

- 本协议不改变各插件的文件访问权限——知识库原始数据层始终只读
- 仅在用户问题明确涉及法律内容时触发，不干扰非法律工作
- 知识库不足时再补充联网搜索或 MCP 工具
- 不得跳过 Step 2 直接检索原始数据源——概念文章是对原始资料的深度综合，应优先使用
- 引用效力警示源时，**必须附效力提示**，且不得作为法律结论的唯一依据
- 概念成熟度 T3/T4 的文章不对外发布，仅供内部消费者（Claude 会话）使用
